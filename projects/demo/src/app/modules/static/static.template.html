Example

When you mouse over an option in a dropdown — it gets focused. You can then use arrows to move focus to another item, keeping mouse hovering over previously focused element.

When you move your cursor — that item is focused again. You can see this behavior in virtually every dropdown menu both in OS and inside browser. Reacting to every mousemove event is unnecessary, because when item is already focused, you do not need to do anything and there is no need to trigger change detection.

@tinkoff/ng-event-plugins allows you to skip events using .silent event modifier and @shouldCall decorator. It also adds .stop and .prevent event modifiers to prevent default and stop propagation in declarative fashion. And .capture modifier to listen to events in capture phase.

HTML and TypeScript source code for Select component

Keep in mind example is heavily simplified. To use AOT all filtering functions must be exported named functions and arguments number must match.

Popup

When Esc button is pressed, popup should close. However, if dropdown is opened in Select — Esc button should only close the dropdown. This can be achieved by stopping event propagation. But if the dropdown is already closed, Select should not handle Esc keydown event. This can be achieved using @shouldCall decorator.
